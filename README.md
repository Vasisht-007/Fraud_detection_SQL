# Fraud_Detection_and_Risk_Analysis_using_SQL

Project Overview
This project analyzes customer account profiles and transaction data using PostgreSQL to identify fraud patterns, evaluate risk factors, and generate actionable insights.

Tools Used
- PostgreSQL
- pgAdmin 4
- SQL

Datasets

Account Profiles
Contains customer account information such as account age, credit limit, risk score, account type, and 2FA status.

Transactions
Contains transaction details including amount, merchant information, device type, transaction risk score, and fraud indicators.

Project Workflow
1. Data Import
2. Data Cleaning
3. Exploratory Data Analysis (EDA)
4. Join Analysis
5. Fraud Pattern Identification
6. Business Recommendations

Key Findings
- Accounts without 2FA experienced higher fraud rates.
- Business accounts showed the highest fraud rate.
- High-risk accounts were more vulnerable to fraud.
- Older accounts generated the highest number of fraudulent transactions.
- Transaction risk scores were strong indicators of fraud.
